本文最后更新于 2021-07-11，文中内容可能已过时。

Kubernetes 是一个自动化的容器编排平台，它负责应用的部署、应用的弹性以及应用的管理，本文介绍使用kubeadm搭建 K8S 集群。

1 环境准备

1.1 主机配置

主机名IP地址操作系统配置
Master10.10.10.60CentOS7.92CPU 2G内存 20G硬盘
Node110.10.10.61CentOS7.92CPU 2G内存 20G硬盘
Node210.10.10.62CentOS7.92CPU 2G内存 20G硬盘

自行安装Docker

以下环境配置需在三台主机都要执行

  1. 关闭selinux
1
2

setenforce 0
sed -i "s/SELINUX=enforcing/SELINUX=disabled/g" /etc/selinux/config
  1. 关闭swap分区或禁用swap文件
1
2

$ swapoff -a
$ sed -ri 's/.*swap.*/#&/' /etc/fstab    # 永久
  1. 关闭防火墙
1
2

[root@master ~]# systemctl stop firewalld
[root@master ~]# systemctl disable firewalld
  1. 修改内核参数
1
2
3
4
5

$ vim /etc/sysctl.conf
net.ipv4.ip_forward = 1
net.bridge.bridge-nf-call-iptables = 1
net.bridge.bridge-nf-call-ip6tables = 1
$ sysctl -p
  1. 启用内核模块
1
2
3
4
5
6

$ vim /etc/sysconfig/modules/ipvs.modules
modprobe -- ip_vs
modprobe -- ip_vs_rr
modprobe -- ip_vs_wrr
modprobe -- ip_vs_sh
modprobe -- nf_conntrack_ipv4
  1. 配置hosts
1
2
3
4

$ vim /etc/hosts
10.10.10.60 master
10.10.10.61 node1
10.10.10.62 node2
  1. 配置时间同步
1
2
3
4
5

# 时间同步
[xdl@k8s-master] ~$ sudo vim /etc/chrony.conf
server ntp.aliyun.com iburst

[xdl@k8s-master] ~$ sudo systemctl restart chronyd.service

重启系统

2 kubernetes集群初始化

Master节点创建集群（该操作只在master主机执行）

1
2
3
4
5
6

$ sudo kubeadm init \
--kubernetes-version=v1.21.2 \
--image-repository registry.aliyuncs.com/google_containers \
--pod-network-cidr=10.244.0.0/16 \
--service-cidr=10.96.0.0/12 \
--apiserver-advertise-address=10.10.10.60

由于默认拉取镜像地址k8s.gcr.io国内无法访问，这里指定阿里云镜像仓库地址。

如果失败，使用kubeadm reset 重置

1
2
3

[xdl@k8s-master] ~$ mkdir -p $HOME/.kube
[xdl@k8s-master] ~$ sudo cp -i /etc/kubernetes/admin.conf $HOME/.kube/config
[xdl@k8s-master] ~$ sudo chown $(id -u):$(id -g) $HOME/.kube/config

node节点加入集群

2.1 kubernetes安装CNI网络插件

kubernetes支持多种网络插件，如：flannel、calico、canal等，任选一种使用即可，本实验选择flannel

只在master节点安装flannel插件即可，该插件使用的是DaemonSet控制器，该控制器会在每个节点上都运行

1
2
3
4
5

#获取flannel配置文件
[root@master ~]# wget https://raw.githubusercontent.com/coreos/flannel/master/Documentation/kube-flannel.yml

#执行文件启动flannel
[root@master ~]# kubectl apply -f kube-flannel.yml
